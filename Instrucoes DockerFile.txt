1 ====> Antes, precisamos do arquivo .jar do projeto.
Para criar o arqvuio .jar, entre na pasta do projeto e execute o seguinte comando:
"./mvnw clean package"
(*Opcional)"
./mvnw package -Pdocker" ====> caso esteja configurado no pom para fazer imagem automaticamente passando argumento, usar este comando
				-P"nome_da_variavel_" ====> tag <id> no pom.xml

Será criado o arquivo .jar na pasta "/target"

2 ====> Parametros do arquivo "Dockerfile":

FROM: Define a imagem base para a imagem que iremos criar. No nosso caso iremos criar uma imagem que necessita do java, portanto seria uma imagem do openjdk. Porém o Openjdk está depreciado
e por isso devemos utilizar como opção a seguinte imagem => eclipse-temurin:17-jre-alpine
Ex. FROM eclipse-temurin:17-jre-alpine

WORKDIR: Define o diretorio de trabalho onde as instruções dos próximos comandos irão rodar . (/app) 
Ex. WORKDIR /app

*Opcional
ARG: esta instrução define uma variavel que pode ser passada em tempo de build. No nosso caso é o argumento "JAR_FILE" que foi definido no plugin do "pom.xml"
Ex. ARG JAR_FILE	

COPY: Recebe uma origem e um destino. A origem é o local onde está o arquivo .jar que será a base da imagem, este arquivo está na pasta "/target/" do projeto.
O destino é o local dentro do container, é o local definido em WORKDIR, seguido do nome do arquivo .jar dentro do container.
Ex. COPY target/*.jar /app/api.jar
EX. COPY target/${JAR_FILE} /app/api.jar ===> se estiver usando JAR_FILE *(Opcional)

EXPOSE: informa qual porta o container irá escutar quando estiver rodado. Ela não publica a porta.
Ex. EXPOSE 8080

CMD: define o comando padrão que será executado quando o container for iniciado. Recomendo usar como array.
Ex. CMD ["java","-jar","api.jar"]

3 ====> Para criar a imagem Docker:
Após criação do .jar e do arquivo "Dockerfile", entrar na pasta do projeto e executar o comando abaixo para a imagem ser criada:
"docker image build -t algafood-api ."

"algafood-api" é o nome da imagem que definirmos.

Resumo: 
./mvnw clean package -Pdocker ===========> cria o JAR do projeto e a imagem no Docker. INFELIZMENTE DEVIDO A LENTIDÃO, PAREI DE USA-LO
./mvnw clean package ====================> cria somente o JAR do projeto
docker image build -t algafood-api . ====> cria a imagem no Docker utilizando o JAR da pasta "/target"
