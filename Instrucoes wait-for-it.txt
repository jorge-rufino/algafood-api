Usamos o wait-for-it quando um serviço depende da total execuçao de outro para que possa funcionar

Link: https://github.com/vishnubob/wait-for-it

1- Copiamos do github todo o código do "wait-for-it.sh"

2- Na pasta raiz do projeto, criamos o arquivo ""wait-for-it.sh" e colamos o código do github.

3- No arquivo "Dockerfile" acrescentamos as linhas abaixo:
COPY wait-for-it.sh /wait-for-it.sh   	=> copia o arquivo .sh para a imagem que será criada no Docker
	
RUN chmod +x /wait-for-it.sh		=> faz com que o arquivo .sh seja executavel

4- No arquivo "docker-compose.yml" acrescentar a linha abaixo depois do atributo "image:"
 command: ["/wait-for-it.sh","algafood-mysql:3306","-t","30","--","java", "-jar", "api.jar"]

=> "command" substitui o "CMD" do Dockerfile. Ele executa o arquivo .sh, e faz com que o servico da API so seja
executado após 30segundos de espera, e somente se o serviço do MySql já estiver finalizado.

==============IMPORTANTE================
Para evitar o erro de bash:

   UTILIZEI ESTA A OPÇÃO 1 POIS A IMAGEM DA API FICA BEM MENOR COM ELE
1- Quando a imagem não vem com o bash instalado, devemos executar um comando para adicionar o bash na imagem
   - eclipse-temurin:17-jre-alpine => tem todos os pacotes inclusive para o jaspereport porém nao tem o bash

Devemos adicionar o comando abaixo para adicionar o bash na imagem:
   - RUN apk add --no-cache bash

2- Utilizar uma imagem que venha com o bash instalado. Abaixo 2 opçoes possiveis
   - openjdk:17-jdk-oracle => pacote completo porém ele deixa a imagem muito grande.
   - openjdk:17-jdk-slim   => pacote maIs enxuto porém ele não contem as bibliotecas para o jaspereport e imagem ainda fica um pouco grande
